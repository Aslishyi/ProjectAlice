# Alice人设扩展规则（非编程人员版）

## 1. 什么是人设？

人设是Alice的性格、背景、兴趣爱好等信息的总和。通过扩展人设，你可以让Alice更加生动、有趣，并且能够更好地与用户进行自然的交流。

## 2. 人设文件介绍

在`app/core/persona/`目录下，有两个主要的人设文件：

### 2.1 extended_persona.json（扩展人设）

这个文件包含了Alice的详细人设信息，如：
- 生活细节（居住环境、日常习惯、兴趣爱好等）
- 知识背景（文学专业知识、阅读偏好等）
- 情感特征（人际关系、情绪表达等）

### 2.2 contextual_persona.json（场景人设）

这个文件包含了Alice在不同场景下的表现特点，如：
- 与陌生人交流
- 与朋友交流
- 与恋人交流

## 3. 如何编辑人设文件？

你可以使用任何文本编辑器（如记事本、VS Code、Sublime Text等）来编辑这些JSON文件。

### 3.1 基本JSON格式规则

JSON文件使用键值对的形式存储数据，基本格式如下：

```json
{
  "键1": "值1",
  "键2": "值2",
  "键3": {
    "子键1": "子值1",
    "子键2": "子值2"
  },
  "键4": ["值1", "值2", "值3"]
}
```

### 3.2 简单的人设扩展方法

你可以按照以下步骤扩展人设：

1. **打开文件**：使用文本编辑器打开`extended_persona.json`文件
2. **定位位置**：找到你想要扩展的部分（如"生活细节"、"兴趣爱好"等）
3. **添加内容**：按照JSON格式添加新的信息
4. **保存文件**：保存你的修改

## 5. 注意事项

1. **格式正确性**：
   - 使用英文双引号（`"`），不要使用中文引号（`“”`）
   - 使用英文逗号（`,`）分隔键值对，不要使用中文逗号（`，`）
   - 确保括号是配对的（`{}`和`[]`）

2. **数据类型**：
   - 文本使用双引号括起来
   - 数字不需要引号
   - 布尔值（true/false）不需要引号
   - 列表使用`[]`括起来，列表中的元素用逗号分隔

3. **简洁明了**：
   - 保持描述简洁，避免过长的句子
   - 使用自然语言，不要使用专业术语

4. **一致性**：
   - 确保新增的人设信息与Alice的核心人设保持一致
   - 避免添加与现有信息矛盾的内容

## 6. 如何验证你的修改？

当你修改完人设文件后，系统会在下次启动时自动将新的人设信息加载到向量数据库中。你可以通过与Alice对话来验证你的修改是否生效。

## 7. 常见问题

### Q: 为什么我的修改没有生效？

A: 可能的原因：
1. JSON格式错误（请检查引号、逗号和括号是否正确）
2. 系统没有重新启动（修改完成后需要重启系统）
3. 新添加的信息与现有信息重复

### Q: 我可以添加哪些类型的人设信息？

A: 你可以添加任何与Alice相关的信息，如：
- 新的兴趣爱好
- 更多的生活细节
- 额外的知识背景
- 特定的情感特征

### Q: 有没有什么内容是不能添加的？

A: 请避免添加以下内容：
- 违法、违规或不适当的内容
- 与Alice核心人设严重矛盾的内容
- 过长、复杂或难以理解的描述
